ISM cert expiry — weekend edge case unhandled High
The compliance module does not handle expiry on a Saturday or Sunday. UAE government offices are closed Friday–Saturday, so the effective renewal deadline is the preceding Thursday. The uaeCompliance.ts rule must detect weekend expiry dates and bring the alert forward by 2–3 days to avoid crew being stranded.

Manual steps — cannot be automated by Claude Code
1
Run all migrations in Supabase SQL editor
In strict order: 001 → 002 → 003 → 004 → 005 → 010 → 011 → 012 → 013 → 014 → 015 → 016 → 017 → 018 → 019. Files are in schema/migrations/. Do not skip any. Migration 012 ALTERs user_profiles from 001 — both must have run successfully first.
2
Enable MFA in Supabase dashboard
Authentication → Sign In / MFA → enable TOTP. This is a project-level toggle — it cannot be set via code or migration. MFA is mandatory for all users. No user should reach any dashboard page without a verified TOTP factor.
3
Deploy Supabase Edge Function for JWT custom claims
An Auth hook Edge Function must write role, org_id, vessel_ids, and module list into the JWT on every login event. Referenced in lib/auth/claims.ts. Deploy via Supabase CLI: supabase functions deploy. Required before any RLS vessel-scope policy will function correctly.
4
Set all environment variables in the hosting platform
ANTHROPIC_API_KEY and SUPABASE_SERVICE_ROLE_KEY are server-only — must never be prefixed NEXT_PUBLIC_. NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY, and NEXT_PUBLIC_APP_URL are safe for the client. Confirm .env.local is in .gitignore before any commit.
5
Configure Supabase Storage bucket for documents
Create a private bucket named vessel-documents. Bucket must be private (not public). All document access uses signed URLs with 1-hour expiry — never expose raw Storage paths. Set max file size and allowed MIME types (PDF, JPG, PNG). Every document view must be audit logged with event_type: 'EXPORT'.
